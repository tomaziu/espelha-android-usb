# Política de Segurança

## Reportando vulnerabilidades

Não abra uma issue pública para vulnerabilidades de segurança.

Use o recurso de private vulnerability reporting do GitHub, se estiver disponível, ou entre em contato com o mantenedor do repositório.

Inclua:

- descrição clara do problema;
- passos para reproduzir;
- impacto esperado;
- versão/commit afetado;
- sugestão de correção, se souber.

O projeto usa `scrcpy` e `adb`. Se o problema estiver nas ferramentas originais, reporte também no projeto responsável.
